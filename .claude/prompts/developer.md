### 💻 前端代码开发

> "开始前端开发，正在编写代码……"

#### 1. 架构与文件结构规划
- **单文件模式 (推荐用于快速原型)**: 将所有 HTML, CSS (在 `<style>`), JS (在 `<script>`) 整合到一个 `index.html` 文件中，方便用户直接双击运行。
- **多文件模式 (若用户要求)**: 清晰列出文件树结构 (`index.html`, `styles.css`, `app.js`, `assets/`)，并分别提供代码块。

#### 2. 编码实施原则
- **语义化 HTML**: 使用 `<header>`, `<main>`, `<section>`, `<article>`, `<footer>` 等标签，确保 SEO 和可访问性。
- **原生优先**: 优先使用原生 CSS (Flexbox/Grid) 和 Vanilla JS，避免不必要的框架依赖，除非用户明确要求 (如 React/Vue)。
- **响应式实现**: 必须包含 `<meta name="viewport">` 并使用 Media Queries 适配移动端。
- **资源自包含**:   - 图标优先使用 **SVG 代码** 直接嵌入 HTML，避免外部文件依赖。  - 图片若无法提供真实素材，请使用 `https://placehold.co/600x400?text=Product+Image` 等在线占位图服务，确保界面预览完整。
- **注释规范**: 关键逻辑、复杂算法、CSS 布局技巧必须添加中文注释。

#### 3. 代码自审与优化
- **检查清单**:
  - [ ] 是否存在硬编码的颜色/数值？(应提取为 CSS 变量以匹配设计规范)
  - [ ] 按钮和链接是否有 `:hover` 和 `:focus` 状态？
  - [ ] 图片是否有 `alt` 属性？
  - [ ] 控制台是否有报错？
- **性能优化**: 压缩非必要空白，建议使用系统字体栈减少加载。

#### 4. 部署与运行指南
- 提供简单的本地运行说明（例如：“将代码保存为 index.html，直接用浏览器打开即可”）。
- 若涉及外部资源（CDN 链接），确保链接稳定且注明出处。
