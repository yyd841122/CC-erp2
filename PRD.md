# 📋 产品需求文档 (PRD)

## 1. 项目背景

- **愿景**: 为小型作坊打造简单易用的 Web 端 ERP 管理系统，实现进销存与财务一体化管理

- **目标用户**:
  - 小型作坊/小微企业主
  - 员工 10 人以内，日单量 < 50 单
  - 年销售额 < 1000 万
  - 主要使用者为企业主本人

- **核心痛点**:
  - 传统进销存软件（如管家婆）操作复杂、不够直观
  - 多台设备无法同时登录操作
  - 缺乏可视化数据分析，经营状况难以快速掌握
  - 期初开账与年终结账流程繁琐

## 3. 用户故事

1. 作为**企业主**，我想要**录入商品信息**，以便于后续进行采购和销售操作

2. 作为**企业主**，我想要**记录采购入库**，以便于库存自动增加和成本核算

3. 作为**企业主**，我想要**记录销售出库**，以便于库存自动扣减和收入统计

4. 作为**企业主**，我想要**查看库存预警**，以便于及时补货避免缺货

5. 作为**企业主**，我想要**查询某产品的历史销售记录和价格**，以便于分析产品销售走势

6. 作为**企业主**，我想要**通过图表查看销售统计**，以便于直观了解经营状况

7. 作为**企业主**，我想要**在2-3台电脑同时登录操作**，以便于多人协作工作

## 4. 业务流程图

### 主流程：
```
期初开账 → 商品/客户/供应商档案建立
   ↓
采购入库 → 库存增加 → 应付账款产生
   ↓
销售出库 → 库存减少 → 应收账款产生 → 收款/付款核销
   ↓
报表查询 → 经营分析
```

### 异常流程：
- **库存不足**: 销售出库时若库存不足，系统提示并阻止操作
- **重复开单**: 同一单据号不允许重复录入
- **数据异常**: 结账前需检查未审核单据，提示用户处理

## 5. 非功能性需求

| 类别 | 要求 |
|:---|:---|
| **性能** | 页面加载时间 < 2s，报表生成 < 3s |
| **兼容性** | 支持主流浏览器（Chrome/Edge/Firefox），支持 Windows 7+ |
| **并发** | 支持 2-3 台电脑同时登录操作 |
| **数据安全** | 本地部署或私有云，数据自主可控，定期自动备份 |
| **易用性** | 界面简洁直观，操作步骤不超过3步完成单据录入 |
